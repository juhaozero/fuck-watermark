# fuck-watermark 项目分析

> 本文档说明项目的核心原理、为何能实现「去水印」、以及当前功能覆盖范围与局限。

---

## 1. 项目是什么

**fuck-watermark** 是一个基于 **Go + Gin** 的短视频/图文 **解析 API 服务**。用户提交各平台的分享链接，服务端代为访问平台页面或官方接口，提取**原始媒体直链**及元数据（标题、作者、封面、音乐等），以统一 JSON 格式返回。

它**不是**视频编辑工具，也**不会**对视频像素做 AI 修复或裁剪处理。

```
用户分享链接 → API 服务 → 模拟浏览器/调用平台接口 → 提取 CDN 直链 → 返回 JSON
```

## 2. 整体架构

```mermaid
flowchart TB
    Client[客户端 / 前端 / 脚本]
    Gin[Gin HTTP 服务]
    MW[中间件链<br/>CORS / 鉴权 / 限流 / SSRF 校验]
    Handler[Handler<br/>/api/parse 或 /api/{platform}]
    Registry[Platform Registry<br/>按域名匹配平台]
    Parser[各平台 Parser]
    HTTP[httputil.Client<br/>出站 HTTP 请求]
    Upstream[平台页面 / 官方 API / CDN]

    Client --> Gin --> MW --> Handler --> Registry --> Parser --> HTTP --> Upstream
    Parser --> Handler
    Handler --> Client
```





### 2.1 核心模块


| 模块       | 路径                                         | 职责                         |
| -------- | ------------------------------------------ | -------------------------- |
| 入口       | `cmd/server/main.go`                       | 加载 `config.toml`，启动服务      |
| 服务组装     | `internal/server/`                         | 注册路由、挂载中间件                 |
| 平台注册表    | `internal/platform/`                       | 7 个内置平台的路由、域名匹配、Parser 工厂  |
| 解析器      | `internal/parser/{platform}/`              | 各平台独立实现 `Parse(ctx, url)`  |
| HTTP 客户端 | `internal/httputil/`                       | 带超时、重定向控制的出站请求             |
| 响应模型     | `internal/model/`                          | 统一 `VideoData` JSON Schema |
| 安全       | `internal/middleware/`、`internal/urlutil/` | 鉴权、限流、SSRF 防护              |




### 2.2 一次请求的完整流程

1. 客户端调用 `GET/POST /api/parse?url=...` 或指定平台路由。
2. **URL 校验**：仅允许 `http/https`，拒绝内网 IP、localhost 等（防 SSRF）。
3. **平台识别**：Registry 按 URL 的 host 后缀（如 `douyin.com`）或关键词匹配 Parser。
4. **Parser 执行**（各平台逻辑不同，见第 4 节）：
  - 短链重定向 → 提取视频/笔记 ID
  - 请求页面 HTML 或官方 API
  - 从 HTML 内嵌 JSON 或 API 响应中解析媒体地址
5. 封装为统一 `VideoData` 结构，返回 `{ code, msg, data }`。

## 3. 为什么能「去水印」



### 3.1 本质：获取无水印 CDN 地址，而非处理视频文件

短视频平台的「水印」在分享场景下，通常通过以下方式实现，而非把水印烧录进唯一一份源文件：


| 手段            | 说明                                  |
| ------------- | ----------------------------------- |
| **不同 CDN 路径** | 同一视频存在「带水印播放地址」与「无水印/低水印地址」，路径或参数不同 |
| **播放器层叠**     | 网页/App 播放时在视频上方叠加水印图层，源文件本身可能无水印    |
| **分享专用流**     | 对外分享返回降质或带标识的流，内部 API 可拿到更高质量源流     |


本项目做的是：**绕过分享页展示给普通用户的地址，去抓平台内部使用的真实媒体 URL**。

### 3.2 典型手段（以抖音为例）

抖音 Parser 从页面内嵌的 `RENDER_DATA` / `_ROUTER_DATA` JSON 中读取 `bitRateList`、`play_addr` 等字段，得到多个 CDN 候选地址，然后：

```go
// 将带水印路径 playwm 替换为无水印路径 play
strings.ReplaceAll(url, "playwm", "play")
```

- `playwm`：Web 分享场景常用的**带水印**播放地址
- `play`：同源的**无水印**播放地址（平台 CDN 本身提供，只是分享页默认不用）

此外还会优先选择 `v3-web` 等高码率线路，并按 `bitRate` 排序取最高清。

### 3.3 其他平台的思路


| 平台       | 去水印/拿直链的方式                                                                                                      |
| -------- | --------------------------------------------------------------------------------------------------------------- |
| **快手**   | 解析页面中 `window.INIT_STATE` / `__APOLLO_STATE__` 内嵌 JSON，直接读取 `mainMvUrls`、manifest 中的 CDN 地址（源站 CDN 直链，分享页不直接暴露） |
| **小红书**  | 从 `__INITIAL_STATE__` 或 discovery API 取 `masterUrl` / `originVideoKey`，拼接 `sns-video-bd.xhscdn.com` 等 CDN       |
| **B 站**  | 调用官方开放接口 `x/web-interface/view` + `x/player/playurl`，获取 `durl[].url` 直链（需 Cookie 时成功率更高）                        |
| **微博**   | POST 官方 `weibo.com/tv/api/component`，从返回的 `urls` 中取多清晰度直链；可选 `proxy_base` 防 403                                 |
| **豆包**   | 调用 `get_video_share_info` 官方分享接口，映射返回中的视频 URL                                                                   |




### 3.4 重要说明

- **「去水印」成功率依赖平台接口与 Cookie 有效性**，平台随时可能改版导致失效。
- **并非所有内容都能去水印**：部分平台对版权内容、付费内容、直播回放等会限制直链获取。
- **B 站等内容**返回的是平台 CDN 直链，是否含平台 Logo/软水印取决于源视频与 CDN 策略，并非本项目额外处理。
- 本项目**不提供**也**不应被用于**绕过版权保护或批量盗用他人作品；请遵守各平台服务条款与著作权法。

## 4. 各平台解析策略详解



### 4.1 抖音（douyin）

```
分享短链 v.douyin.com
    → 重定向跟随
    → 正则提取 modal_id / video_id
    → GET douyin.com/user/self?modal_id={id}
    → 解析 RENDER_DATA 或 _ROUTER_DATA
    → 选最高 bitRate + playwm→play
```

支持：短视频、图文笔记、图集；Live Photo（`live_photo` 字段）。

### 4.2 快手（kuaishou）

```
分享链接
    → 重定向
    → 识别 short-video / long-video / photo
    → 抓取页面 HTML
    → 解析 INIT_STATE 或 __APOLLO_STATE__
    → 提取 mainMvUrls / manifest / atlas 图集
```

支持：短视频、长视频、单图、图集（atlas）。

### 4.3 小红书（xiaohongshu）

```
xhslink / 站内链接
    → 重定向 + 提取 note_id
    → 页面 __INITIAL_STATE__
    → fallback: discovery/item API（需 xsec_token）
    → h265/h264 流按码率排序
```

支持：视频笔记、图文笔记、Live Photo。

### 4.4 哔哩哔哩（bilibili）

```
BV 号 / b23.tv 短链
    → 官方 view API 取 cid 列表
    → 逐 P 调用 playurl API（qn=112 高画质）
    → 返回 parts[] 多 P 结构
```

支持：单 P / 多 P 视频；高清建议配置 Cookie。


### 4.5 微博（weibo）

```
weibo.com/tv / video.weibo.com / t.cn 短链
    → 递归重定向提取 fid
    → POST tv/api/component
    → 多清晰度 urls + 可选 proxy_base 代理
```

支持：2K / 1080P / 720P 等多档，`video_backup` 含 label/quality。

### 4.6 豆包 AI 视频（doubao）

```
video-sharing 链接（share_id + video_id）
    → POST get_video_share_info
    → 映射 video_info 字段为 VideoData
```

## 5. 覆盖范围



### 5.1 已支持平台


| 平台   | API 路由                           | 识别域名/关键词                                 | 媒体类型             |
| ---- | -------------------------------- | ---------------------------------------- | ---------------- |
| 抖音   | `/api/douyin`                    | `douyin.com`, `v.douyin.com`             | 视频、图集、Live Photo |
| 快手   | `/api/kuaishou`                  | `kuaishou.com`, `chenzhongtech`, `kspkg` | 视频、长视频、图集、单图     |
| 小红书  | `/api/xiaohongshu`, `/api/xhsjx` | `xiaohongshu.com`, `xhslink.com`         | 视频、图文、Live Photo |
| 哔哩哔哩 | `/api/bilibili`                  | `bilibili.com`, `b23.tv`                 | 视频（含多 P）         |
| 微博   | `/api/weibo`                     | `weibo.com`, `t.cn`                      | 视频（多清晰度）         |
| 豆包   | `/api/doubao`                    | `doubao.com`                             | AI 分享视频          |
| 自动识别 | `/api/parse`                     | 上述全部                                     | 按 URL 自动路由       |




### 5.2 支持的链接形式

- 各平台**分享短链**（如 `v.douyin.com`、`b23.tv`、`t.cn`）
- 各平台**完整页面 URL**（含 query 参数）
- 请求方式：`GET` / `POST`（表单 `url=` 或 JSON `{"url":"..."}`）



### 5.3 统一返回字段（VideoData）


| 字段                              | 说明                                   |
| ------------------------------- | ------------------------------------ |
| `platform`                      | 平台标识                                 |
| `type`                          | `video` / `image` / `live` / `mixed` |
| `title`, `desc`, `cover`, `url` | 基础元数据与主媒体地址                          |
| `author`                        | `{ name, id, avatar }`               |
| `video_backup`                  | 备用清晰度/线路                             |
| `parts`                         | B 站等多 P 视频                           |
| `images`, `live_photo`          | 图集与 Live Photo                       |
| `music`                         | 背景音乐信息                               |
| `stats`                         | 点赞、播放等（部分平台）                         |
